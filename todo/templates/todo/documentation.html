Dodawanie użytkowników

Wymaga uprawnień: canMakeUsers

Składnia: localhost:8000/api/user/{parametr},{wartość},{parametr2},{wartość2}.../{appkey}
Przykład: localhost:8000/api/user/type,add,name,testuser,password,5d9c68c6c50ed3d02a2fcf54f63993b6/testappkey-111222

Parametry mogą być pisane w dowolnej kolejności, a po każdym parametrze musi zostać umieszczona powiązana z nim wartość.
Wymagane parametry:

  • type(add) - określa typ działania na użytkownikach, w przypadku dodawania przyjmuje wartość 'add'
  • name('nazwa użytkownika') - określa nazwę użytkownika
  • password('hasło') - określa hasło użytkownika [UWAGA: hasło powinno zostać wysłane w postaci zaszyfrowanej przez md5]

appkey - klucz aplikacji podany przez administratora podczas rejestracji aplikacji

Odpowiedź serwera w postaci pliku JSON zawiera:

  • Result(Success|Error) - status zapytania
  • [Result==Error] Error('opis błedu') - opis błędu zapytania

Wymaga uprawnień: canMakeNotes

Składnia: localhost:8000/api/note/{parametr},{wartość},{parametr2},{wartość2}.../{appkey}
Przykład: localhost:8000/api/note/type,add,title,tytul,content,tekstnotatki,importance,3,token,user223344/testappkey-111222

Parametry mogą być pisane w dowolnej kolejności, a po każdym parametrze musi zostać umieszczona powiązana z nim wartość.
Wymagane parametry:

  • importance(0|1|2|3) - waga notatki
  • token('token użytkownika') - unikalny token użytkownika, można go uzyskać poprzez logowanie do api
  • content('opis') - dodatkowy opis notatki [jeżeli brak proszę jako wartośc umieścić 'null']
